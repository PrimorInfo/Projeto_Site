Afinal, o que é Git e Github?

O GitHub é uma espécie de rede social de desenvolvedores. Enquanto o Git é um software, o GitHub é uma plataforma onde você pode subir seus projetos, criar seu portfólio e compartilhá-lo publicamente.

O versionamento pode ser um arquivo word, tabela excel e etc. O reflexo dele no VSCode, é fantástico.


Git

Git é um sistema de controle de versão de arquivos. Através deles podemos desenvolver projetos na qual diversas pessoas podem contribuir simultaneamente no mesmo sistema, editando e criando novos arquivos e permitindo que os mesmos possam existir sem o risco de suas alterações serem sobrescritas.

Se não houveesse versionamento, imagine o caos quando duas pessoas abrisse o mesmo arquivo ao mesmo tempo para implementações, por exemplo. Uma das aplicações do git é justamente essa, permitir que um arquivo possa ser editado ao mesmo tempo por desenvolvedores diferentes. Por mais complexo que isso seja, ele tenta manter tudo em ordem para evitar problemas e conflitos insolúveis.

Outro fator importante do git é a possibilidade de criar, a qualquer momento, vários snapshots do seu projeto, ou como chamamos: branches. Suponha que o seu projeto seja um site html, e você deseja criar uma nova seção no seu código HTML, mas naquele momento você não deseja que estas alterações estejam disponíveis, ou seja, você quer alterar o projeto, mas ainda não quer que isso seja tratado como “oficial” para outras pessoas, então você cria um branch (como se fosse uma cópia espelho) e então trabalha apenas nesse branch, até acertar todos os detalhes dele. Após isso, fazer um merge de volta do seu branch até o projeto original.


Github

O Github é um serviço web que oferece diversas funcionalidades extras aplicadas ao git. Resumindo, você poderá usar gratuitamente o github para hospedar seus projetos pessoais. Além disso, quase todos os projetos, frameworks e bibliotecas sobre desenvolvimento open source estão no github, e você pode acompanhá-los através de novas versões, contribuir informando bugs ou até mesmo enviando código e correções.

Se você é desenvolvedor e ainda não tem git e github, você está atrasado e essa é a hora de levar os seus conhecimentos para o próximo nível.
