Afinal, o que é Git e Github?

O GitHub é como uma rede social de desenvolvedores. Enquanto o Git é um software, o GitHub é uma plataforma onde você pode subir seus projetos, criar seu portfólio e compartilhá-lo publicamente.

O versionamento pode ser um arquivo word, tabela excel e etc. O reflexo dele no VSCode, é fantástico.


Git

Git é um sistema de controle de versão de arquivos. Através deles podemos desenvolver projetos na qual diversas pessoas podem contribuir simultaneamente no mesmo sistema, editando e criando novos arquivos e permitindo que os mesmos possam existir sem o risco de suas alterações serem sobrescritas.

Se não houvesse versionamento, imagine o caos quando duas pessoas abrisse o mesmo arquivo ao mesmo tempo para implementações, por exemplo. Uma das aplicações do git é justamente, permitir que um arquivo possa ser editado ao mesmo tempo por desenvolvedores diferentes. Por mais complexo que isso seja, ele tenta manter tudo em ordem para evitar problemas e conflitos insolúveis.

Outro fator importante do git é a possibilidade de criar, a qualquer momento, várias versões de um projeto, denominadas: branches.


Github

O Github é um serviço web que oferece diversas funcionalidades extras aplicadas ao git. O github é uma forma para hospedar projetos pessoais de forma pública ou privada. Além disso, quase todos os projetos, frameworks e bibliotecas sobre desenvolvimento open source estão no github, e pode-se acompanhá-los através de novas versões, contribuir informando bugs ou até mesmo enviando código e correções.

Um desenvolvedor que ainda não utiliza git e github, está atrasado e passou da hora de levar os seus conhecimentos para o próximo nível.
